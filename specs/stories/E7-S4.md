# E7-S4: Parse/Edit Page with TipTap Split View

**Epic:** E7 — Frontend Core
**Layer:** UI
**Group:** D
**Depends on:** E7-S1, E2-S3

## Description

Document parse and edit page with split view: original file info on the left, TipTap rich text editor on the right loaded with parsed markdown. Supports parse trigger, content editing, and saving.

## Acceptance Criteria

1. Parse page at /documents/:id/parse renders split view: document info panel (left) and TipTap editor (right)
2. TipTap editor configured with StarterKit + Table extensions, loads parsed markdown content from GET /api/v1/parse/:id/content
3. "Parse Document" button triggers POST /api/v1/parse/:id with loading spinner; disabled if document is already parsed
4. "Save Edits" button sends PUT /api/v1/parse/:id/content with editor content, transitions status to edited
5. "Proceed to Classify" button visible when document status is parsed or edited, navigates to /documents/:id/classify
