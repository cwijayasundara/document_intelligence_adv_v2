# E5-S3: Extraction API + Results Repository + Review Gate

**Epic:** E5 — Extraction & Confidence Judging
**Layer:** Repository + API
**Group:** D
**Depends on:** E5-S1, E5-S2, E2-S2

## Description

Extraction API that orchestrates extractor -> judge flow, persists results, and enforces a hard review gate. Users must review all low-confidence fields before the document can proceed to summarization.

## Acceptance Criteria

1. Extraction results repository provides async save_results, get_by_document_id, update_values operations for extracted_values table
2. POST /api/v1/extract/:id runs extractor subagent then judge subagent, saves all results with confidence scores to extracted_values
3. GET /api/v1/extract/:id/results returns extraction results: list of {field_name, display_name, extracted_value, source_text, confidence, confidence_reasoning, requires_review, reviewed}
4. PUT /api/v1/extract/:id/results accepts updated values for specific fields, sets reviewed=true for those fields
5. Hard review gate: document status cannot transition to extracted if any extracted_value has requires_review=true AND reviewed=false
6. Returns 400 with list of unreviewed fields if review gate fails
