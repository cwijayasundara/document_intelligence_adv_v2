# E9-S2: Bulk Job Repository + API

**Epic:** E9 — Bulk Processing
**Layer:** Repository + API
**Group:** E
**Depends on:** E9-S1, E1-S4

## Description

Persistence layer and API for bulk processing jobs. Supports creating jobs, tracking per-document progress, and querying job status.

## Acceptance Criteria

1. Bulk job repository provides async create, get_by_id, list_all, update_status for bulk_jobs and bulk_job_documents tables
2. POST /api/v1/bulk/upload accepts multiple files, creates a bulk_job with status=pending and individual bulk_job_documents records, starts pipeline processing in background
3. GET /api/v1/bulk/jobs returns list of bulk jobs with: id, status, total_documents, processed_count, failed_count, created_at
4. GET /api/v1/bulk/jobs/:id returns job details with per-document breakdown: document_id, file_name, status, error_message, processing_time_ms
5. Failed documents have error_message populated with the failure reason from the pipeline
