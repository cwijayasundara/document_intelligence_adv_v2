# E7-S2: Dashboard — Document List with Status

**Epic:** E7 — Frontend Core
**Layer:** UI
**Group:** C
**Depends on:** E7-S1, E2-S1

## Description

Dashboard page at / showing all documents in a table with status badges, category, file info. Each row links to the next action based on document status. Empty state for new users.

## Acceptance Criteria

1. Dashboard at / renders a table with columns: file_name, status, category, file_type, file_size, created_at
2. DocumentStatusBadge component renders color-coded badges: uploaded=gray, parsed=blue, edited=blue, classified=yellow, extracted=orange, summarized=purple, ingested=green
3. Each row links to the appropriate next-action page based on status (e.g., uploaded → /documents/:id/parse, classified → /documents/:id/extract)
4. Empty state displayed when no documents exist, with a call-to-action link to /upload
5. Document list auto-refreshes via TanStack Query with 30-second refetch interval
