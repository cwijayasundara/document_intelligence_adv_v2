# E3-S4: Long-Term Memory (PostgreSQL-Backed)

**Epic:** E3 — Agent Framework & Memory
**Layer:** Repository + Service
**Group:** B
**Depends on:** E1-S1, E1-S3

## Description

Persistent memory backed by PostgreSQL for conversation summaries and generic key-value storage. Adapted from reference backend's PostgresLongTermMemory, stripped of multi-tenancy.

## Acceptance Criteria

1. Alembic migration creates conversation_summaries and memory_entries tables
2. save_conversation_summary() persists session_id, agent_type, summary text, key_topics, documents_discussed, queries_count with upsert semantics
3. get_conversation_summary(session_id) retrieves summary for a given session
4. Generic key-value store: put(namespace, key, data), get(namespace, key), delete(namespace, key), search(namespace) with JSONB data column
5. All long-term memory operations are async and decorated with retry logic for transient database errors
