# E8-S2: Classification Page with Override

**Epic:** E8 — Frontend AI & Config
**Layer:** UI
**Group:** E
**Depends on:** E7-S1, E4-S3

## Description

Classification results page showing the detected category with reasoning. Allows manual override via dropdown. Enforces state machine (only accessible for parsed/edited documents).

## Acceptance Criteria

1. Classification page at /documents/:id/classify displays detected category name and reasoning text
2. "Classify" button triggers POST /api/v1/classify/:id with loading state
3. Category override dropdown shows all available categories; selecting a different category updates documents.document_category_id
4. "Accept & Proceed" button saves classification and navigates to /documents/:id/extract
5. Page shows 404 or redirect if document status is not parsed or edited
