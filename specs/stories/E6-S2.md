# E6-S2: Weaviate Client + Semantic Chunking + Ingestion

**Epic:** E6 — Summarization & RAG
**Layer:** Service + API
**Group:** C
**Depends on:** E1-S2, E2-S1

## Description

Weaviate integration for vector storage and hybrid search. Includes collection setup, LangChain SemanticChunker for intelligent document splitting, and ingestion API that chunks and upserts documents with metadata.

## Acceptance Criteria

1. Weaviate client connects to local Docker instance using WEAVIATE_URL from config
2. DocumentChunks collection created on startup with text2vec-openai vectorizer and hybrid search enabled
3. LangChain SemanticChunker splits parsed markdown on semantic boundaries with fallback to 512 tokens max, 100 token overlap
4. POST /api/v1/ingest/:id chunks the document's parsed content and upserts to Weaviate with metadata: document_id, document_name, document_category, file_name, chunk_index, created_at
5. Re-ingestion: if chunks exist for document_id, deletes all existing chunks before re-inserting
6. Document status transitions to ingested after successful ingestion
