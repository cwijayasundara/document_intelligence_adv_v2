# E2-S1: Document Repository + Upload Service + API

**Epic:** E2 — Document Upload & Parsing
**Layer:** Repository + Service + API
**Group:** B
**Depends on:** E1-S1, E1-S3, E1-S4

## Description

Full vertical slice for document management: repository CRUD, upload service with SHA-256 dedup, and REST API endpoints for upload, list, get, and delete operations.

## Acceptance Criteria

1. Document repository provides async create, get_by_id, list_all, delete operations using SQLAlchemy async sessions
2. POST /api/v1/documents/upload accepts multipart file upload, saves file to upload_dir, creates document record with status=uploaded, and returns document details
3. SHA-256 file dedup: if uploaded file hash matches an existing document, return the existing document record instead of creating a duplicate
4. GET /api/v1/documents returns list of all documents with fields: id, file_name, status, document_category_id, file_type, file_size, created_at
5. GET /api/v1/documents/:id returns full document details including all fields
6. DELETE /api/v1/documents/:id removes document record from database and deletes associated files from upload_dir and parsed_dir
