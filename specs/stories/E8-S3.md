# E8-S3: Extraction Results 3-Column View + Review Gate

**Epic:** E8 — Frontend AI & Config
**Layer:** UI
**Group:** E
**Depends on:** E7-S1, E5-S3

## Description

3-column extraction results table with confidence badges, inline editing for values, and a hard review gate that prevents proceeding until all low-confidence fields are reviewed.

## Acceptance Criteria

1. Extraction page at /documents/:id/extract renders 3-column table: Field Name (+ confidence badge), Extracted Value, Source Text
2. Confidence badges color-coded: green/high, yellow/medium, red/low with tooltip showing confidence_reasoning
3. Low-confidence rows highlighted with amber background, "Requires Review" label, and inline "Edit" button
4. Inline editing: clicking Edit on a field opens an input; saving sends PUT /api/v1/extract/:id/results and marks field as reviewed
5. "Save & Proceed to Summary" button disabled with message "Review all flagged fields first" until all requires_review=true fields have reviewed=true
6. "Extract" button triggers POST /api/v1/extract/:id with loading state; shows results when complete
