# E4-S1: Category & Extraction Schema CRUD

**Epic:** E4 — Classification & Config Management
**Layer:** Repository + API
**Group:** B
**Depends on:** E1-S1, E1-S3, E1-S4

## Description

Full CRUD for document categories and extraction field schemas. Categories define classification targets with criteria text. Extraction fields define what to extract per category. Seeds a default "Other/Unclassified" category on first run.

## Acceptance Criteria

1. GET /api/v1/config/categories returns all categories with id, name, description, classification_criteria
2. POST /api/v1/config/categories creates a new category with name, description, classification_criteria and returns the created record
3. PUT /api/v1/config/categories/:id updates category fields and returns updated record
4. DELETE /api/v1/config/categories/:id deletes category; returns 400 if documents are assigned to it
5. GET /api/v1/config/categories/:id/fields returns extraction fields for the category ordered by sort_order
6. POST /api/v1/config/categories/:id/fields accepts a list of field definitions (field_name, display_name, description, examples, data_type, required, sort_order) and creates/updates them
7. Default "Other/Unclassified" category seeded automatically if no categories exist on app startup
