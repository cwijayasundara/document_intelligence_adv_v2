# E5-S1: Extractor Subagent with Dynamic Pydantic Models

**Epic:** E5 — Extraction & Confidence Judging
**Layer:** Service
**Group:** C
**Depends on:** E3-S1, E3-S2, E4-S1

## Description

DeepAgent subagent that extracts structured fields from classified documents. Dynamically builds a Pydantic model from the document category's extraction field schema. Uses DeepAgent response_format for structured output.

## Acceptance Criteria

1. Extractor registered as a DeepAgent subagent with tools: get_extraction_schema (loads fields for category), get_parsed_content (reads parsed markdown)
2. Dynamic Pydantic model built at runtime from extraction_fields: each field becomes a model attribute with appropriate type (str, float, date based on data_type)
3. Extractor returns ExtractionResult with list of ExtractedField(field_name, extracted_value, source_text) for each schema field
4. PII middleware applied before the extraction LLM call
5. Structured output enforced via DeepAgent response_format parameter using the dynamic Pydantic model
