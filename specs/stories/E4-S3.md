# E4-S3: Classification API Endpoint

**Epic:** E4 — Classification & Config Management
**Layer:** API
**Group:** D
**Depends on:** E4-S2, E2-S2

## Description

REST endpoint that triggers document classification via the classifier subagent. Enforces state machine (document must be parsed or edited). Saves classification result to database.

## Acceptance Criteria

1. POST /api/v1/classify/:id triggers classifier subagent on the document
2. Response includes category_id, category_name, reasoning
3. Classification result saved to documents.document_category_id
4. Document status transitions to classified via state machine
5. Returns 400 if document status is not parsed or edited
