# E1-S2: Application Configuration

**Epic:** E1 — Foundation Infrastructure
**Layer:** Config
**Group:** A
**Depends on:** —

## Description

Centralized configuration loading from config.yml (storage paths, chunking params, bulk concurrency, RAG defaults) and .env (API keys, database URL). Exposed as typed Pydantic settings objects for type-safe access throughout the application.

## Acceptance Criteria

1. config.yml loaded with storage paths (upload_dir, parsed_dir, schemas_dir), chunking parameters (max_tokens, overlap_tokens), bulk concurrency setting, and RAG defaults (default_search_mode, default_alpha, top_k)
2. .env loaded with OPENAI_API_KEY, REDUCTO_API_KEY, DATABASE_URL, WEAVIATE_URL, OPENAI_MODEL
3. Configuration accessible as typed Pydantic settings objects with validation
4. Missing required environment variables raise a clear error message at startup with the variable name
5. Default values provided for non-secret config: upload_dir="./data/upload", parsed_dir="./data/parsed", max_tokens=512, overlap_tokens=100
