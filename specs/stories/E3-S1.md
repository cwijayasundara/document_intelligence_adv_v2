# E3-S1: DeepAgent Orchestrator Scaffold

**Epic:** E3 — Agent Framework & Memory
**Layer:** Service
**Group:** B
**Depends on:** E1-S2

## Description

Set up the DeepAgents orchestrator using create_deep_agent() with GPT-5.4-mini. Register placeholder subagents for classifier, extractor, judge, summarizer, and RAG retriever. Include FilesystemMiddleware and SubAgentMiddleware. Provide singleton factory access.

## Acceptance Criteria

1. DeepAgent orchestrator created via create_deep_agent() with model="openai:gpt-5.4-mini"
2. Subagent registry configured with 5 named subagent slots: classifier, extractor, judge, summarizer, rag_retriever (initially stubs)
3. Middleware stack includes FilesystemMiddleware, SubAgentMiddleware, and SummarizationMiddleware
4. Agent factory singleton provides thread-safe access to the orchestrator instance via async lock
5. Orchestrator initializes without error and responds to a basic health-check prompt
