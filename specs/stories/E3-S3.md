# E3-S3: Short-Term Memory (Session-Based)

**Epic:** E3 — Agent Framework & Memory
**Layer:** Service
**Group:** C
**Depends on:** E3-S1

## Description

In-memory session-based conversation history for agent interactions. Uses LRU-like trimming to manage context window. Supports per-session message tracking for RAG chat and other multi-turn interactions.

## Acceptance Criteria

1. ShortTermMemory class provides per-session message storage with configurable max_messages (default 20)
2. Messages automatically trimmed when exceeding max_messages: preserves system message + most recent messages
3. Supports add_human_message, add_ai_message, get_messages, clear_session, delete_session operations per session_id
4. get_conversation_summary(session_id) returns formatted text of last 6 messages (3 exchanges)
5. get_session_count() returns total number of active sessions
