# E2-S3: Reducto Parser Integration + Parse/Edit API

**Epic:** E2 — Document Upload & Parsing
**Layer:** Service + API
**Group:** C
**Depends on:** E2-S1

## Description

Reducto Cloud API client for parsing documents (PDF, DOCX, XLSX, images) to markdown. API endpoints for triggering parse, retrieving parsed content, and saving edited content. Includes dedup optimization to skip re-parsing unchanged files.

## Acceptance Criteria

1. Reducto client sends document file to Reducto Cloud API and receives markdown output with tables and structure preserved
2. Parsed markdown saved to parsed_dir/{document_id}.md and document.parsed_path updated in database
3. POST /api/v1/parse/:id triggers parsing via Reducto, transitions document status to parsed, returns parsed markdown content
4. GET /api/v1/parse/:id/content returns the parsed markdown content from the filesystem
5. PUT /api/v1/parse/:id/content accepts edited markdown, saves to parsed_dir, transitions status to edited
6. Parsing skipped if document.file_hash is unchanged and parsed file already exists — returns existing parsed content with 200
